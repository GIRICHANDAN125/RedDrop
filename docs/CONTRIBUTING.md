# Contributing

Thank you for improving Red Drop AI. The repository is still under active development, so contributions should focus on small, safe changes that keep the current behavior intact.

## Before You Start

- Read the README and setup guide.
- Check the existing folder structure before adding new files.
- Avoid introducing secrets, generated files, or unrelated refactors.

## Workflow

1. Create a descriptive branch name.
2. Install dependencies in `backend/` and `frontend/`.
3. Make the smallest possible change that solves the problem.
4. Validate the touched area locally.
5. Open a pull request with a clear summary.

## Code Style

- Keep the existing JavaScript style and folder boundaries.
- Do not modify business logic unless the task requires it.
- Prefer documentation, configuration, and incremental improvements for non-feature work.

## Pull Request Checklist

- [ ] The change is scoped and easy to review.
- [ ] Secrets are not committed.
- [ ] Documentation is updated when needed.
- [ ] The repository still runs locally.
